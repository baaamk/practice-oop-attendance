# 출석부 기능구현

- 출석 시스템의 출석 확인은 크루가 캠퍼스에 들어온 후 시스템에 출석 데이터가 저장된 시간을 기준으로 한다.
- 시간은 24시간 형식만 사용한다. 예를 들어, "22:30"은 오후 10시 30분을 의미한다.
- 교육 시간은 월요일은 13:00~18:00, 화요일~금요일은 10:00~18:00이다.
    - 5분 초과는 지각으로 간주한다.
    - 30분 초과는 결석으로 간주한다.
    - 출석 기록이 없는 날은 결석으로 간주한다.
- 누적 지각 및 결석 횟수에 따라 경고 또는 면담을 시행한다.
    - 지각 3회는 결석 1회로 간주한다.
        - 경고 대상자: 결석 2회 이상
        - 면담 대상자: 결석 3회 이상
        - 제적 대상자: 결석 5회 초과
- 캠퍼스 운영 시간은 매일 08:00~23:00이다.
- 주말 및 공휴일에는 출석을 받지 않는다.
- 출석 시스템에 등록된 크루와 12월 출석 기록은 제공된 파일(attendances.csv)에서 확인할 수 있다.
- 프로그램은 사용자가 종료할 때까지 종료되지 않으며, 해당 기능을 수행한 후 초기 화면으로 돌아간다.

---

### 출석 flow

이름 입력 → 날짜 고정. 시각 입력

### 출석 수정 flow

이름 입력 → 출석 수정 날짜(일) 입력 → 변경 시각 입력 → 전  후 출력 

### 출석 기록 확인 flow

이름 입력 → 일별 출석 기록 출력 → 유형별 총 개수 출력 → 대상 상태 출력

### 제적 위험자 확인 flow(경고 이상 전부 출력)

결과만 출력 

```
빙티: 결석 3회, 지각 4회 (면담)
- 이든: 결석 2회, 지각 5회 (면담)
- 빙봉: 결석 1회, 지각 6회 (면담)
- 쿠키: 결석 2회, 지각 3회 (면담)
- 짱수: 결석 0회, 지각 6회 (경고)
```

# 기능

- 출석확인, 출석수정, 크루별 출석 기록 확인, 제적 위험자 확인, 종료
- 프론트 어댑터 구현
    - 출석확인 컨트롤러
    - 출석수정 컨트롤러
    - 크루별 출석 기록 확인 컨트롤러
    - 제적 위험자 확인 컨트롤러
- 출석부 클래스
    - 파일 읽기
    - 필드: 크루
- 크루 클래스(일급 컬렉션)
    - 필드: 이름, 시간, 상태(결석, 지각, 면담 or 경고)
- 제적 위험자 클래스(?)
    - 필드: 크루
- 서비스 레이어
- view를 어떻게 나눌지
- DTO 만들기(전체적으로)
